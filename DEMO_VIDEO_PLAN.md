# 導覽影片產出計畫（待處理，先擱置）

目標流程：模擬註冊 → 選擇站點並開始 → 山海兩條路線 → 減碳點數兌換 → 個人頁面 → VIP 專屬內容與付費訂閱

## 已確認的技術決策

- **產生方式**：用 Playwright 寫自動化腳本，本機啟動瀏覽器實際點過整個流程並錄成 `.webm`/`.mp4`，存在本機專案目錄下。需要額外安裝 Playwright（devDependency）並下載一份 Chromium。
- **註冊驗證碼處理**：「模擬註冊」畫面只照樣錄輸入 email、看到驗證碼畫面的過程；驗證碼那一步無法真的收信自動輸入，改用一組**已存在的測試帳號**直接登入，接續完成後面的流程，影片看起來仍是完整走一遍。

## 待補資訊（真的要動手時需要跟使用者確認）

1. **測試帳號**：需要一組 email/使用者名稱 + 密碼的既有帳號，且帳號狀態要適合展示（有騎乘紀錄、有足夠點數可以示範兌換、等級最好接近 VIP／master tier 讓 profile 頁面好看）。若沒有現成的，需要先用先前 admin 帳號建立方式（直接寫 SQL insert `auth.users`/`profiles`）造一個 demo 帳號。
2. **本機 dev server**：錄影腳本要跑在 `npm run dev` 起的本機伺服器上，跑之前要確認本機 `.env.local` 的 Supabase 連線是可用的（能不能連到真的專案，或需要另外一份測試用 Supabase 專案）。
3. **影片解析度／長度／要不要加字幕或旁白** 目前沒特別要求，先當作單純畫面操作錄製，不加旁白。

## 預計錄製步驟（對應腳本要點的每一段）

1. **模擬註冊**：`/login` 頁「註冊」分頁 → 輸入 email → 顯示「輸入驗證碼」畫面（示意即可）→ 切換成用測試帳號登入，導向首頁。
2. **選擇站點並開始**：首頁 GO 按鈕 → `/ride` 選擇起始站點 → 開始騎乘畫面。
3. **山海兩條路線**：`/route` 頁面，切換海線／山線 segment，展開幾個站點時間軸。
4. **減碳點數兌換**：`/redeem` 頁，展示直接兌換一個「永續好禮」的完整流程（含成功彈窗）。
5. **個人頁面**：`/profile`，展示等級徽章、成就格、統計數字。
6. **VIP 專屬內容與付費訂閱**：`/route` 的 VIP 主題路線卡片展開 → 點「更多內容」進 `/route/unlock` → 展示單次解鎖／訂閱方案卡片，選一個走到「感謝贊助」畫面。

## 明確擱置事項

這份計畫先寫下來存底，**這次不執行**，使用者要求先處理其他問題。之後要動手時，先確認上面「待補資訊」三項，再開始寫 Playwright 腳本。
